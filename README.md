# Executive Summary

Customer churn directly impacts recurring revenue, customer lifetime value, and long-term business growth. Every customer lost represents not only lost revenue but also increased acquisition costs.

**Churn Radar** is an executive analytics platform designed to help business leaders identify churn drivers, monitor customer health, estimate revenue at risk, and support proactive retention strategies through interactive business intelligence.

The platform transforms raw customer data into actionable insights, enabling faster and more informed business decisions.

---

# Business Problem

SaaS organizations continuously face challenges such as:

- Increasing customer churn
- Declining customer engagement
- Rising customer acquisition costs
- Revenue leakage
- Limited visibility into customer behavior

Without continuous monitoring, organizations react **after** customers leave instead of preventing churn before it happens.

---

# Business Objective

The objective of Churn Radar is to provide executives and business managers with a centralized analytics platform capable of:

- Monitoring customer retention
- Identifying churn trends
- Measuring customer engagement
- Tracking revenue performance
- Supporting customer success initiatives
- Enabling data-driven decision making

---

# Solution Overview

Churn Radar provides an integrated executive dashboard that combines customer analytics, business intelligence, and operational reporting into a single platform.

The application enables organizations to:

- Monitor customer churn
- Analyze customer behavior
- Track customer engagement
- Measure revenue at risk
- Evaluate subscription performance
- Generate business insights
- Support retention planning

# Installation

Clone the repository

```bash
git clone https://github.com/Akshith1704/A1-Churn-Radar-SaaS-Customer-Churn-Analytics.git
```

Move into the project

```bash
cd A1-Churn-Radar-SaaS-Customer-Churn-Analytics
```

Install dependencies

```bash
npm install
```

Run locally

```bash
npm run dev
```

Build production version

```bash
npm run build
```

Preview production build

```bash
npm run preview
```
